# Video Assembler API 🎬🔧

![image.png](https://i.postimg.cc/9M9B5n5m/image.png)

Проект представляет собой API для автоматической сборки видеороликов из предоставленных пользователем материалов. Это **"Видео-Конструктор"**: вы даете ему изображения, видео, текст и сценарий, а он автоматически "склеивает" всё в готовое видео с озвучкой, музыкой и субтитрами.

**Ключевая философия проекта — "Принеси свои медиа" (Bring Your Own Media).** Система не использует внешние AI-сервисы для генерации контента, что делает её работу на 100% предсказуемой, стабильной и бесплатной в использовании.

## 🌟 Ключевые возможности

-   **Сборка по сценам:** Создание сложных видео из набора ваших картинок и видеофрагментов с точными таймингами.
-   **TTS-озвучка:** Автоматическая генерация голосовой дорожки из предоставленного текста (русский женский голос).
-   **Выбор фоновой музыки:** Вы можете выбрать один из нескольких треков, которые заранее добавлены в проект.
-   **Автоматические субтитры:** Система может сама распознать сгенерированную речь и создать идеально синхронизированные субтитры.
-   **Асинхронная работа:** Все тяжелые задачи (сборка, рендеринг) выполняются в фоновом режиме, не заставляя вас ждать ответа от API.

## 🏗️ Архитектура

Проект — это система из нескольких Docker-сервисов, каждый из которых выполняет свою роль:

| Сервис | Роль | Описание |
| :--- | :--- | :--- |
| **`backend`** | Менеджер | FastAPI-приложение, которое принимает API-запросы, проверяет данные и создает "заказ" на сборку видео. |
| **`worker`** | Рабочий | Celery-воркер, который выполняет всю тяжелую работу: генерирует аудио, общается с CapCut, собирает сцены. |
| **`redis`** | Диспетчер | Хранит очередь "заказов" для воркера и статусы их выполнения. |
| **`capcut-mate`** | Монтажер | Open-source API, которое управляет видеоредактором CapCut для фактической "склейки" видео. |
| **`linux` (Nginx)** | Кладовщик | Простой веб-сервер, который отдает вам готовые видеофайлы для скачивания. |

## 🚀 Быстрый старт

### Требования

-   [Docker](https://www.docker.com/get-started) и Docker Compose.
-   Git для клонирования репозитория.

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/bossnikitos1k/avatar-generation.git
cd [НАЗВАНИЕ_ПАПКИ_ПРОЕКТА]
```

### Шаг 2: (Опционально) Добавление своей музыки

Вы можете добавить свои собственные музыкальные треки для выбора.

1.  Найдите папку `backend/assets/`.
2.  Положите в нее ваши `.mp3` файлы (например, `my-cool-track.mp3`).
3.  Название файла (без `.mp3`) вы сможете использовать в API.

### Шаг 3: Сборка и запуск проекта

Проект не требует API-ключей. Вы можете сразу его запустить.

```bash
docker-compose up --build -d
```

Эта команда соберет все сервисы и запустит их в фоновом режиме. Первая сборка может занять несколько минут.

### Шаг 4: Использование API

1.  **Откройте интерактивную документацию (Swagger UI)** в браузере:
    [http://localhost:8000/docs](http://localhost:8000/docs)

2.  **Откройте `POST /api/generate`** и нажмите **"Try it out"**.

3.  **Заполните поля:**

| Параметр | Описание | Пример |
| :--- | :--- | :--- |
| **`image`** | **(Обязательно)** Главное изображение-аватар. | `(выберите файл)` |
| **`scene_media`** | **(Обязательно)** Все картинки и видео для сцен. | `(выберите несколько файлов)` |
| **`text`** | **(Обязательно)** Полный текст для голосовой озвучки. | `"Привет, это тестовое видео..."` |
| **`scenes_json`** | **(Обязательно)** JSON-сценарий с таймингами. | `[{"start_time": 0, "text": "Первая сцена"}]` |
| **`use_background_music`**| Включить фоновую музыку? | `true` |
| **`music_track`** | Название трека из папки `assets`. | `"background"` или `"uplifting"` |
| **`add_subtitles`** | Включить авто-субтитры? | `true` |

4.  Нажмите **"Execute"**. В ответ вы получите `task_id`.

### Шаг 5: Проверка статуса и скачивание

-   **Проверить статус задачи** (замените `{task_id}` на ваш):
    `GET http://localhost:8000/api/status/{task_id}`

-   **Скачать готовое видео** (когда статус `SUCCESS`):
    Перейдите по ссылке из ответа о статусе или напрямую по адресу:
    `http://localhost/api/download/{task_id}`
    *(Обратите внимание: для скачивания используется порт `80`, а не `8000`)*.

## 🔧 Отладка

-   **Просмотр логов:**
    ```bash
    # Следить за логами "рабочего", который собирает видео
    docker-compose logs -f worker

    # Посмотреть логи "менеджера", который принимает запросы
    docker-compose logs -f backend
    ```

-   **Полная остановка системы:**
    ```bash
    docker-compose down
    ```
